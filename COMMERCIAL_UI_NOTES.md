# DataShield BY 2.3 Commercial UI

## Что изменено
- Полностью обновлён Web UI в коммерческом стиле.
- Добавлен рабочий сценарий:
  1. вход в систему,
  2. загрузка SQLite-базы,
  3. запуск маскирования одной кнопкой,
  4. вывод результата,
  5. сохранение маскированной базы,
  6. отображение точного пути сохранения и ссылки на скачивание.
- Маскирование выполняется безопасно на копии исходной базы.
- Сгенерированный YAML-конфиг сохраняется автоматически в `storage/configs/`.
- Исходные загруженные базы сохраняются в `storage/uploads/`.
- Маскированные базы сохраняются в `storage/masked/`.

## Ограничение текущего UI-потока
Прямой upload workflow в этой версии поддерживает SQLite-файлы:
- `.db`
- `.sqlite`
- `.sqlite3`

Для PostgreSQL / MySQL / Oracle остаются штатные backend-конфиги и job flow через YAML.

## Ключевые API
- `POST /api/databases/upload`
- `POST /api/databases/mask`
- `GET /api/databases`
- `GET /api/databases/download/<filename>`

## Где искать результат
- Загруженная база: `storage/uploads/`
- Маскированная база: `storage/masked/`
- Автосгенерированный конфиг: `storage/configs/`
